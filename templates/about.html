Secquence Diagram


How a user can upload an image to a website and get various outputs based on the image's caption is illustrated here. The website uses an “ImageCaptioningModel” to process the image and generate a caption in English. The website displays the caption to the user. The user can choose to convert the caption to voice speech or another language using “GoogleAPI”. The website plays the voice speech or displays the translated text to the user. The diagram shows the messages exchanged between the user, the website, the “ImageCaptioningModel” and the “GoogleAPI”, as well as the activation and return values of each entity. The diagram helps to understand the flow of events and the interactions involved in the process.

Mathematical Model


Users upload an image, which is then encoded into a format the model understands. The core of the system, a trained image caption model, analyzes the encoded image and generates a corresponding caption. This caption is then decoded back into human-readable text and presented to the user. Additional features like voice playback and translation enhance user experience, making image descriptions readily accessible.

DFD Level 3


The diagram outlines a process for generating and translating text captions from images. The user starts by browsing an input image from their local device. A deployed model then generates a caption using different methods (Greedy, Beam (3), or Beam (5)). The generated caption can be viewed directly, or it can be sent to a voice generator for voice reading. Additionally, there is a translation function that utilizes the API of Google Translate to provide multi-language translations of the generated text caption.
